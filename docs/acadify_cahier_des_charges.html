Résumé exécutif

Acadify est une application mobile multi-plateforme développée par Codify, conçue pour centraliser et structurer la communication et la gestion académique au sein de l'UIECC. Elle remplace les canaux informels (WhatsApp, PDF partagés) par un outil officiel, sécurisé et optimisé pour le contexte numérique camerounais.

🎯
Problème résolu
Absence de canal institutionnel structuré. Tout passe par WhatsApp : communiqués, notes, absences, cours. Information fragmentée et inaccessible.
CORE PROBLEM
📱
Solution proposée
Application Flutter avec backend Firebase. Notifications push, bibliothèque offline, notes et emploi du temps interactif en temps réel.
MVP PHASE 1
🌍
Vision long terme
Déploiement scalable dans d'autres universités de la CEMAC. Modèle gratuit pour l'institution, maintenance facturée par Codify.
CEMAC SCALE
⚡
Différenciateur clé
Mode hors-ligne natif — cours, EDT et notes accessibles sans data. Critique pour Sangmélima où la connectivité reste intermittente.
OFFLINE FIRST
Positionnement Codify : Acadify est offert gratuitement à l'UIECC en phase initiale. Codify conserve la propriété intellectuelle et assure la maintenance. L'université bénéficie d'une solution de transformation numérique sans investissement ; Codify construit une référence client institutionnelle et un produit scalable vers d'autres universités.
02

Contexte & Problématique

L'UIECC, pôle d'excellence en génie numérique, se retrouve dans un paradoxe : une université qui forme des ingénieurs du numérique mais dont la communication interne repose entièrement sur WhatsApp.

Situation actuelle
  • ✗Communiqués officiels diffusés dans des groupes WhatsApp non officiels
  • ✗Notes et résultats partagés en PDF/Word bruts sans organisation ni historique
  • ✗Cours transmis via les délégués de classe — chaîne humaine fragile
  • ✗Aucun annuaire structuré des enseignants et services administratifs
  • ✗Étudiants sans compte WhatsApp ou sans data exclus de l'information
Contexte favorable
  • ✓Centre d'incubation numérique inauguré en 2025 à Sangmélima
  • ✓Partenariat KL University (Inde) — programmes informatique actifs
  • ✓Vocation tech affirmée de l'ESIGN — écosystème réceptif à l'innovation
  • ✓Financement FODEC/CEMAC actif — institution en croissance
  • ✓0 application mobile officielle existante — marché vierge
Insight clé : Le problème n'est pas WhatsApp en lui-même, c'est l'absence d'alternative institutionnelle structurée. WhatsApp s'est imposé par défaut. Pour le déloger durablement, Acadify doit être objectivement supérieur sur tous les points qui expliquent son usage actuel : notifications, facilité de partage, accessibilité. Et apporter en plus ce que WhatsApp ne peut pas offrir : organisation, recherche, offline, rôles, sécurité des données académiques.

04

Périmètre fonctionnel

PHASE 1 · MVP Fonctionnalités de base Mois 1–2
  • Communiqués officiels — Publication admin, push notification instantanée, archivage consultable
  • Bibliothèque de cours — Upload PDF/DOCX par enseignants, organisé par matière et semestre, offline
  • Consultation des notes — Affichage sécurisé par étudiant, historique semestres, moyenne automatique
  • Suivi des absences — Nombre par matière, seuil d'alerte configurable, notification auto
  • Emploi du temps — Vue semaine interactive, modification admins, rappel 15 min avant cours
  • Annuaire campus — Contacts enseignants, délégués, services admin — filtrables et recherchables
  • Authentification sécurisée — Connexion matricule + mot de passe, rôles Étudiant / Enseignant / Admin
  • Mode hors-ligne — Cache local des cours téléchargés, EDT et notes via Hive/SQLite
PHASE 2 · ENRICHISSEMENT Fonctionnalités avancées Mois 3–4
  • Messagerie directe — Canal enseignant → étudiant(s), groupes par filière, pas de chat libre entre étudiants
  • Dashboard enseignant — Statistiques consultation des cours, taux d'ouverture des communiqués
  • Notifications avancées — Paramétrage par catégorie, mode silencieux planifiable
  • Recherche globale — Recherche full-text dans les cours, communiqués et annuaire
  • Profil étudiant — Photo de profil, filière, niveau, informations de contact éditables
  • Statistiques de présence — Vue graphique de l'évolution des absences par matière sur le semestre
PHASE 3 · INSTITUTIONNALISATION Maturité & Scalabilité Mois 5–6
  • Dashboard admin web — Interface Flutter Web : gestion étudiants, cours, notes, communiqués, rôles
  • Support site Ouesso — Extension au campus congolais avec isolation des données par site
  • Paiement frais scolaires — Intégration Orange Money / MTN MoMo — consultation solde et historique
  • Internationalisation — Support FR / EN (bilingue Cameroun + étudiants anglophones)
  • API ouverte — Documentation REST permettant l'intégration avec les systèmes de l'université
  • Portail web companion — Version navigateur pour les étudiants sans smartphone récent

Architecture technique

Stack sélectionné pour sa performance, sa faisabilité solo en phase initiale, et sa scalabilité future vers d'autres universités.

FRONTEND MOBILE
Flutter 3.x (Dart)
Android & iOS — 1 codebase
Hive + SQLite (cache offline)
Material Design 3
flutter_bloc (gestion d'état)
go_router (navigation)
BACKEND / SERVICES
Firebase Firestore (BDD)
Firebase Authentication
Firebase Storage (fichiers)
FCM (push notifications)
Cloud Functions (triggers)
Security Rules granulaires
ADMINISTRATION
Flutter Web (admin panel)
Rôles : Admin / Prof / Étudiant
Upload cours & documents
Gestion notes & absences
Logs d'activité admin
Dashboard statistiques
⚡ Stratégie Offline-First : Toutes les données critiques (cours téléchargés, EDT actuel, dernières notes) sont synchronisées localement via Hive au premier accès avec connexion. En mode hors-ligne, l'app sert les données du cache. Les actions nécessitant internet sont mises en file d'attente et exécutées à la reconnexion de façon transparente.

Sécurité des données

Authentification
Connexion par matricule + mot de passe haché. Tokens JWT Firebase avec expiration. Pas de connexion via réseaux sociaux pour garantir la confidentialité académique.
Isolation des données
Rules Firestore garantissant qu'un étudiant ne voit que ses propres notes et absences. Un enseignant n'accède qu'aux données de ses matières et filières.
Chiffrement
Toutes les communications en HTTPS/TLS. Fichiers Firebase Storage non accessibles sans token authentifié valide. Données locales Hive chiffrées avec clé unique par appareil.

Contraintes & Exigences

📶
Connectivité limitée
Sangmélima présente une connectivité mobile intermittente. L'application doit fonctionner en mode offline complet pour toutes les fonctionnalités de consultation (cours, EDT, notes). La synchronisation s'opère à la reconnexion, de façon transparente et silencieuse.
📱
Appareils cibles
Priorité absolue Android (85%+ du parc étudiant en Afrique centrale). Compatibilité minimum : Android 8.0 (API 26). iOS : support secondaire iPhone SE+. L'app ne doit pas dépasser 40 Mo. Optimisation appareils entrée de gamme (2 Go RAM minimum).
🌐
Langue et localisation
Interface principale en français. Support anglais dès la Phase 3 (étudiants anglophones du Cameroun et du Congo). Format dates : JJ/MM/AAAA, heure 24h. Normalisation UTC + affichage local.
💾
Stockage et performance
Quota Firebase Firestore free tier (1 GiB stockage, 50K lectures/jour) à respecter en Phase 1. Fichiers PDF limités à 20 Mo par pièce. Compression automatique des images uploadées. Cache local limité à 500 Mo par appareil.
🔒
Confidentialité des données académiques
Les notes et absences sont des données personnelles sensibles — jamais visibles d'autres étudiants. Conformité RGPD basique : possibilité de suppression du compte, pas de partage avec tiers. Logs d'activité anonymisés après 90 jours.
⚙️
Maintenabilité long terme
Code documenté en français et anglais. Architecture modulaire — chaque fonctionnalité est un module isolé. Tests unitaires pour les fonctions critiques (auth, affichage notes). README permettant à un autre développeur Flutter de reprendre le projet en moins de 2 jours.

Modèle de collaboration

La relation Codify / UIECC est structurée comme un partenariat gagnant-gagnant, pas comme une simple donation.

Codify apporte
  • Développement complet de l'application (Phase 1 offerte)
  • Hébergement Firebase Phase 1 (quota free tier)
  • Maintenance corrective 6 mois post-lancement
  • Mises à jour sécurité sur toute la durée du partenariat
  • Formation de 1–2 étudiants ESIGN comme mainteneurs internes
  • Documentation technique complète du projet
  • Branding Codify intégré discrètement dans l'app
UIECC apporte
  • Accès aux données nécessaires (structures cours, filières, EDT)
  • Désignation d'un référent admin interne (1 enseignant ou staff)
  • Communication officielle encourageant l'adoption étudiante
  • Lettre de recommandation / référence institutionnelle pour Codify
  • Accord de principe pour les phases 2 et 3 (développements payants)
  • Visibilité du projet dans les événements de l'université
Modèle économique Phase 2+ : À partir de la Phase 2, les nouvelles fonctionnalités et l'augmentation de quota Firebase font l'objet d'un devis Codify. Tarification suggérée : forfait maintenance annuel 300 000–500 000 FCFA + développement sur devis. Ce modèle garantit qu'Acadify dure 20 ans indépendamment du statut étudiant du fondateur.
