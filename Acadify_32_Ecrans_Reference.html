Acadify — Référence des 32 Écrans · Phase 1

MVP · Flutter · Firebase
🔵 Écrans transversaux — tous les rôles
01Splash ScreenALL
  • Logo Acadify centré + animation de chargement
  • Vérification du token de session en arrière-plan
  • Redirection auto vers Connexion ou Dashboard selon statut
  • Fond couleur principale #0B6E4F, durée ~2s
02OnboardingALL
  • 3 slides de présentation (affiché uniquement au 1er lancement)
  • Slide 1 : "Tous tes cours en un endroit" + icône livre
  • Slide 2 : "Tes notes en temps réel" + icône résultats
  • Slide 3 : "Accessible sans internet" + icône wifi barré
  • Bouton "Commencer" sur la dernière slide → Connexion
  • Indicateur de progression (points) + bouton "Passer"
03ConnexionALL
  • Logo Acadify + nom de l'université en haut
  • Champ : Numéro matricule
  • Champ : Mot de passe (avec bouton afficher/masquer)
  • Bouton "Se connecter" (primary, pleine largeur)
  • Lien "Mot de passe oublié ?" en bas
  • Gestion erreur : mauvais identifiants
04Mot de passe oubliéALL
  • Champ : Numéro matricule pour identification
  • Instructions simples pour contacter l'administration
  • Bouton retour vers Connexion
  • Message de confirmation après soumission
05Première connexionALL
  • Affiché uniquement si compte créé par admin (mot de passe provisoire)
  • Message : "Bienvenue ! Créez votre mot de passe personnel"
  • Champ : Nouveau mot de passe
  • Champ : Confirmer le mot de passe
  • Règles de sécurité affichées (min 6 caractères)
  • Bouton "Confirmer et accéder"
🟢 Rôle Étudiant — 17 écrans
06Dashboard / AccueilÉTUDIANT
  • Salutation personnalisée : "Bonjour, [Prénom] 👋"
  • Carte "Prochain cours" : matière, heure, salle
  • Section "Derniers communiqués" (2–3 aperçus)
  • Section "Mes notes récentes" (2–3 dernières)
  • Indicateur absences du jour (badge rouge si proche seuil)
  • Barre de navigation basse : Accueil / Cours / Notes / Planning / Profil
07Liste des communiquésÉTUDIANT
  • Liste chronologique des communiqués officiels
  • Chaque item : titre, date, badge type (Urgent / Info / Général)
  • Indicateur "non lu" (point coloré)
  • Filtre par catégorie en haut (tabs ou chips)
  • Tap sur un item → Détail communiqué
08Détail d'un communiquéÉTUDIANT
  • Titre complet en gros
  • Date de publication + auteur (Administration / Scolarité)
  • Badge de catégorie (Urgent / Info)
  • Corps du communiqué (texte complet, scrollable)
  • Pièce jointe si applicable (bouton télécharger PDF)
  • Bouton retour vers la liste
09Bibliothèque de coursÉTUDIANT
  • Sélecteur de semestre en haut (Semestre 1 / 2)
  • Liste des matières du semestre actif
  • Chaque matière : nom, enseignant, nombre de documents disponibles
  • Badge "Nouveau" si document récemment ajouté
  • Indicateur offline (icône téléchargé si disponible hors-ligne)
  • Tap → Liste documents de la matière
10Documents d'une matièreÉTUDIANT
  • Nom de la matière en titre
  • Liste des fichiers : titre, date upload, taille, type (Cours / TD / Exam)
  • Bouton télécharger (offline) ou lire sur chaque item
  • Indicateur de téléchargement en cours (progress bar)
  • Badge "Offline" si déjà téléchargé
11Visionneuse PDFÉTUDIANT
  • Affichage du PDF en plein écran (scrollable page par page)
  • Barre supérieure : titre du document + bouton fermer
  • Bouton télécharger offline (si pas encore téléchargé)
  • Indicateur de page actuelle / total pages
  • Zoom pinch-to-zoom supporté
12Mes notesÉTUDIANT
  • Sélecteur semestre en haut
  • Carte "Moyenne générale" bien visible en haut
  • Liste des matières avec note obtenue + note max + coefficient
  • Code couleur : vert (validé) / orange (moyen) / rouge (insuffisant)
  • Tap sur matière → Détail note
13Détail d'une noteÉTUDIANT
  • Nom de la matière + enseignant
  • Note obtenue / Note maximale bien mis en avant
  • Coefficient et part dans la moyenne générale
  • Commentaire du professeur (si renseigné)
  • Date de publication de la note
14Mes absencesÉTUDIANT
  • Résumé global : total absences / seuil maximum autorisé
  • Barre de progression visuelle (vert → rouge selon remplissage)
  • Liste par matière : nb absences / nb séances total
  • Alerte rouge si seuil critique atteint dans une matière
  • Date de chaque absence (en tap sur la matière)
15Emploi du tempsÉTUDIANT
  • Navigation par semaine (← semaine précédente / suivante →)
  • Vue grille Lundi → Vendredi avec créneaux horaires
  • Chaque cours : matière, salle, enseignant, couleur par matière
  • Cours actuel mis en surbrillance
  • Tap sur un cours → Détail créneau
  • Fonctionne hors-ligne (données en cache)
16Détail d'un créneauÉTUDIANT
  • Nom de la matière (titre)
  • Enseignant responsable
  • Salle / Bâtiment
  • Horaire précis (début → fin)
  • Type de séance : Cours Magistral / TD / TP
  • Statut : Normal / Annulé / Reporté
17Annuaire campusÉTUDIANT
  • Barre de recherche en haut (nom, matière)
  • Filtres : Enseignants / Délégués / Administration
  • Chaque item : photo (avatar si absent), nom, fonction
  • Tap → Fiche contact détaillée
18Fiche contactÉTUDIANT
  • Photo / Avatar + nom complet + titre/fonction
  • Matières enseignées (si enseignant)
  • Filière / Niveau représenté (si délégué)
  • Email et/ou numéro de bureau (si renseigné)
  • Heures de disponibilité (si renseignées)
19Mon profilÉTUDIANT
  • Photo de profil (avatar par défaut) + nom complet
  • Matricule, filière, niveau, année d'inscription
  • Statistiques rapides : nb cours téléchargés, nb absences totales
  • Bouton "Modifier" (Phase 2)
  • Bouton "Se déconnecter"
20ParamètresÉTUDIANT
  • Toggle notifications globales On/Off
  • Toggle notifications communiqués
  • Toggle notifications notes
  • Toggle notifications EDT (rappel avant cours)
  • Gestion du cache offline (voir taille, vider)
  • Version de l'application + mentions légales
21Mode hors-ligneÉTUDIANT
  • Bannière discrète en haut : "Mode hors-ligne — données du [date]"
  • Contenu disponible en cache : cours téléchargés, EDT, notes
  • Sections non disponibles grisées avec icône info
  • Bouton "Synchroniser" dès que la connexion revient
22État vide (liste vide)ÉTUDIANT
  • Illustration simple centrée (pas de données)
  • Message contextuel selon la section : "Aucun cours disponible", "Aucun communiqué"…
  • Bouton d'action si pertinent : "Actualiser" ou "Retour"
  • Réutilisé sur toutes les listes vides de l'app
🟡 Rôle Enseignant — 5 écrans
23Dashboard enseignantPROF
  • Nom de l'enseignant + liste de ses matières
  • Actions rapides : Uploader cours / Saisir notes / Saisir absences
  • Résumé : nb de cours uploadés, dernière mise à jour EDT
  • Accès au profil enseignant
  • Barre de navigation : Dashboard / Mes matières / Notes / Absences
24Uploader un coursPROF
  • Sélecteur de matière (liste déroulante)
  • Sélecteur de type : Cours / TD / TP / Examen
  • Champ titre du document
  • Bouton sélectionner fichier (PDF, DOCX depuis le téléphone)
  • Barre de progression d'upload
  • Confirmation : "Cours publié — les étudiants ont été notifiés"
25Saisir les notesPROF
  • Sélecteur matière + sélecteur évaluation (Partiel 1 / DS / Exam final…)
  • Note maximale configurable
  • Liste de tous les étudiants de la filière avec champ note
  • Calcul automatique de la moyenne de classe en bas
  • Bouton "Enregistrer et publier" (notifie tous les étudiants)
26Saisir les absencesPROF
  • Sélecteur matière + date de la séance
  • Liste des étudiants avec toggle Présent / Absent pour chacun
  • Compteur de présents / absents en temps réel
  • Option "Marquer tous présents" pour aller vite
  • Bouton "Valider l'appel"
27Mettre à jour l'EDTPROF
  • Vue de ses créneaux de la semaine
  • Tap sur un créneau : modifier salle / horaire / statut
  • Statut : Normal / Annulé / Reporté (avec nouveau créneau)
  • Motif d'annulation (optionnel)
  • Confirmation : notification automatique aux étudiants concernés
🟣 Rôle Administrateur — 5 écrans
28Dashboard adminADMIN
  • Statistiques globales : nb étudiants actifs, nb profs, nb communiqués publiés
  • Actions rapides : Nouveau communiqué / Ajouter utilisateur
  • Fil des dernières activités (qui a uploadé quoi, quand)
  • Indicateur santé système (Firebase / notifications)
  • Barre de navigation : Dashboard / Communiqués / Utilisateurs
29Créer un communiquéADMIN
  • Champ titre du communiqué
  • Sélecteur catégorie : Urgent / Information / Général
  • Éditeur corps du texte (multi-lignes)
  • Ciblage : Toute l'université / Par filière / Par niveau
  • Option pièce jointe (PDF)
  • Bouton "Publier" → notification push envoyée immédiatement
30Gérer les communiquésADMIN
  • Liste de tous les communiqués publiés (titre, date, catégorie)
  • Statut : Publié / Archivé
  • Swipe ou bouton pour modifier / supprimer / archiver
  • Filtre par catégorie ou date
  • Bouton "+" pour créer un nouveau communiqué
31Gestion des utilisateursADMIN
  • Liste de tous les utilisateurs (nom, matricule, rôle, filière)
  • Barre de recherche par nom ou matricule
  • Filtre par rôle : Étudiant / Enseignant / Admin
  • Tap sur un utilisateur : voir profil, modifier rôle, désactiver compte
  • Bouton "+" pour ajouter un utilisateur
32Ajouter un utilisateurADMIN
  • Champ : Numéro matricule
  • Champ : Nom complet
  • Sélecteur rôle : Étudiant / Enseignant / Admin
  • Sélecteur filière + niveau (si étudiant)
  • Sélecteur matières enseignées (si enseignant)
  • Mot de passe provisoire généré automatiquement
  • Bouton "Créer le compte" → compte actif immédiatement
